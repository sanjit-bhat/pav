what are the benefits of wish_getNextEp?
- wish_getNextep is basically a logical translation of the checks
the getNextEp func runs, along with the determ input->output computation.
- the benefits all boil down to: it bundles the checks
and determinism reasoning together, which makes the proof easier,
especially if you use wish_getNextEp in a wrapper Server postcond.
- e.g.,
in wp_getNextEp:
in err=true case, need to say that one of checks failed.
in err=false case, need to say that all checks passed.
wish_getNextEp de-duplicates the checks across the err branches.
- e.g.,
when getNextEp rets err, in Blame proof,
need to argue that Server passes all the checks.
this is easier when Server postcond bundles checks into wish_getNextEp.
- [most interesting] e.g.,
when doing check on getNextEp output, in Blame proof,
need to argue that Server passes the check.
this is easier when apply wish_getNextEp_det lemma to
align Client computation with Server postcond.
this hides the chain of determ internal to getNextEp.
